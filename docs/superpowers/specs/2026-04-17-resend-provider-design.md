# Design — terraform-provider-resend

**Date:** 2026-04-17  
**Status:** Approved  
**Registry:** `realianx/resend`  
**Module:** `github.com/realianx/terraform-provider-resend`

---

## Overview

A public Terraform provider for the [Resend](https://resend.com) API, built with `hashicorp/terraform-plugin-framework`. Mirrors the structure of `terraform-provider-dokploy` in the same monorepo.

---

### Client package (`resend/`)

- Base URL: `https://api.resend.com`
- Auth: `Authorization: Bearer <api_key>`
- `User-Agent` header set to `terraform-provider-resend/<version>`
- `HTTPError` type with `StatusCode` for 404 detection (resource removed from state)
- `do(ctx, method, path, body, out)` shared by all domain files

### Provider config

```hcl
provider "resend" {
  api_key = "re_xxx"  # or env: RESEND_API_KEY
}
```

Only `api_key` — no host (Resend has a fixed base URL).

## Resources

### `resend_template`

**API:** `POST/GET/PATCH/DELETE /templates/{id}`

**Schema:**

| Attribute   | Type     | TF     | Notes |
|-------------|----------|--------|-------|
| id          | string   | computed | UUID |
| name        | string   | required | |
| html        | string   | required | |
| alias       | string   | optional | |
| from        | string   | optional | |
| subject     | string   | optional | |
| reply_to    | string   | optional | |
| text        | string   | optional | plain-text version; omit to let Resend auto-generate from HTML |
| published   | bool     | optional | default `false`; triggers `POST /templates/{id}/publish` when set to `true` |

**Behaviour:** On Create or Update, if `published=true`, call publish endpoint after the write.  
Setting `published=false` after a previously published template is a no-op — Resend has no unpublish endpoint; the attribute only controls whether to trigger a publish call.

---

### `resend_segment`

**API:** `POST/GET/DELETE /segments/{id}` (no update endpoint)

**Schema:**

| Attribute | Type   | TF       | Notes |
|-----------|--------|----------|-------|
| id        | string | computed | UUID |
| name      | string | required | ForceNew — API has no update |

**Behaviour:** Name change destroys and recreates the segment.

---

### `resend_topic`

**API:** `POST/GET/PATCH/DELETE /topics/{id}`

**Schema:**

| Attribute            | Type   | TF       | Notes |
|----------------------|--------|----------|-------|
| id                   | string | computed | UUID |
| name                 | string | required | max 50 chars |
| default_subscription | string | required | `opt_in` or `opt_out`; ForceNew (immutable per API) |
| description          | string | optional | max 200 chars |
| visibility           | string | optional | `public` or `private`; default `private` |

---

### `resend_automation`

**API:** `POST/GET/PATCH/DELETE /automations/{id}`, `POST /automations/{id}/stop`

**Schema:**

| Attribute   | Type   | TF       | Notes |
|-------------|--------|----------|-------|
| id          | string | computed | UUID |
| name        | string | required | |
| status      | string | optional | `enabled` or `disabled`; default `disabled` |
| steps       | string | optional | JSON string (`jsonencode(...)`) |
| connections | string | optional | JSON string (`jsonencode(...)`) |

**Behaviour:** `steps` and `connections` must be provided together (API requirement).  
Stored as raw JSON strings in state; validated at apply time by the API.

---

### `resend_event`

**API:** `POST/GET/PATCH/DELETE /events/{id}`

**Schema:**

| Attribute | Type   | TF       | Notes |
|-----------|--------|----------|-------|
| id        | string | computed | UUID |
| name      | string | required | cannot use `resend:` prefix |
| schema    | string | optional | JSON string of flat key/type pairs |

---

## Error Handling

- 404 responses → remove resource from state (not an error)
- Non-2xx → `Diagnostics.AddError` with status + body
- `HTTPError` type used for type-asserting 404s (same pattern as Dokploy provider)

---

## Publishing

- goreleaser: multi-platform binaries (linux/darwin/windows × amd64/arm64), GPG signed
- GitHub Actions: `test.yml` on PR/push, `release.yml` on `v*` tags
- Secret required: `GPG_PRIVATE_KEY`
- Registry source: `realianx/resend`

---

## Out of Scope

- Data sources (read-only lookups) — not requested
- Contacts, Audiences, API keys, Domains — not requested
- Step/Connection typed blocks for automations — JSON raw string chosen for flexibility
